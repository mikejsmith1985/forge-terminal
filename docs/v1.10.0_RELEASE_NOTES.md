## ğŸš€ How to Use

### Opening Files
1. Look at the left sidebar - you'll see two tabs: **Files** and **Cards**
2. Click the **Files** tab (ğŸ“ icon)
3. Browse the file tree - click folders to expand/collapse
4. **Double-click any file** to open it in the editor
5. Or **right-click** â†’ "Open in Editor"

### Editing Files
1. File opens in **right panel** (40% width)
2. Edit code with full **syntax highlighting**
3. Press **Ctrl+S** or click **Save** to save changes
4. Click **X** to close the editor
5. Modified files show a **â— dot** indicator

### Context Menu Actions
**Right-click files**:
- Open in Editor
- Send to Terminal (pastes `./filename`)
- Copy Path (full path)
- Copy Relative Path
- Delete

**Right-click folders**:
- Open in Terminal (executes `cd folder && ls`)
- Send Path to Terminal
- New File (coming soon)
- New Folder (coming soon)
- Delete

### Quick Tips
- **Double-click folders** to expand/collapse
- **Double-click files** to open in editor
- **.gitignored files** appear dimmed and italic
- **Executable files** (.js, .py, .go, .sh) have a **Run** button

## ğŸ”§ Technical Details

### Dependencies Added
- `@monaco-editor/react` - VS Code editor component
- `react-resizable-panels` - For future panel resizing

### Bundle Size
- Frontend: **879KB** (up from ~650KB)
- Monaco editor adds ~230KB gzipped
- Lazy-loaded on first file open

### Performance
- File tree: **Max depth 3** (configurable)
- File read: **10MB limit** (configurable)
- Tree auto-collapses: `node_modules`, `.git`, etc.
- .gitignore parsing: **Simple glob matching**

### Security
- **Path validation** prevents directory traversal
- **File size limits** prevent memory issues
- **Same-origin only** - no remote file access
